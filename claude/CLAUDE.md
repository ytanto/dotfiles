# CLAUDE.md

このファイルは、コードを扱う際のClaude Code (claude.ai/code) への指針を提供します。

## ガイドライン

- 指示者に対しては日本語で回答すること
- 質問に対して勝手に行動を開始しないこと。まずは回答すること。
- 自動テストが導入されている場合は、
  - コードの修正後は必要に応じてテストコードを追加すること
  - コードの修正後は必ず自動テストを実行し、全てのテストがパスすることを確認すること
  - テストが失敗した場合は、修正を行い再度テストを実行すること
- コードの修正後は不要・利用されていないファイルやコードを削除すること
- 常にMVPを意識し、必要な機能のみを実装すること
- **後方互換が崩れる変更・既存機能の挙動が変わる変更を行う前は必ず指示者に確認すること**
  - 既存の条件分岐 / 権限制御 / 動的ロジック / API 仕様 / props・emit などを削除・置換するときは、変更後の挙動と影響範囲を明示して GO サインを取る
  - 「見た目を寄せる」「リファクタする」等の依頼でも、内部仕様が変わるなら確認する。スタイル統合の依頼で機能仕様を勝手に削除・固定値化しない
- 開発やレビューでリポジトリを横断する場合は、対象リポジトリの CLAUDE.md や rules を明示的に読むこと
  - これらは実行ディレクトリのツリー外にあると自動では読み込まれないため、横断時は能動的に参照する
- GitHubを参照する場合は、ghコマンドを優先して利用すること
- コミットメッセージに「レビュー対応」等、コードから読み取れないコンテキストの情報を含めないこと
  - あくまでコードをどういった目的・背景・内容を変更したかをメッセージとすること
- サーバーサイドの実装は必ずTDDで行うこと

### TypeScript

- interfaceは可能な限り利用せず、typeを使用すること
- useEffectの多用は避け、シンプルなロジックを心がけること
