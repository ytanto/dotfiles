# テストケース生成（シンプル）

**出力形式**: $1
**対象機能**: $ARGUMENTS

## 方針

- 1観点1行で簡潔に。説明を盛らない
- 各カテゴリ最大5項目程度。網羅より要点
- 正常系 / 異常系 / 境界値 のみ。サブセクションを切らない
- 自明な観点（押せる・表示される等）は省く

## 出力フォーマット

### list

```markdown
### 正常系
- [ ] 〇〇できる
- [ ] 〇〇が維持される

### 異常系
- [ ] 〇〇 → エラー表示

### 境界値
- [ ] 〇〇 → エラーにならない
```

### table

```markdown
| 分類 | 対象 | 期待結果 |
|------|------|----------|
| 正常系 | 〇〇 | 〇〇 |
```

## 出力ルール

- コードブロックで囲まずそのまま出力（コピペ可）
- 前置き・補足説明は書かない
