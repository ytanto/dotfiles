# テストケース生成（マークダウン形式）

指定された機能のテストケースをマークダウン形式で生成してください。

## 引数

- 第1引数（`$1`）: 出力形式 - `table`（表形式）または `list`（箇条書き）
- 第2引数以降（`$2` 以降）: 機能名・機能の説明

**出力形式**: $1
**対象機能**: $ARGUMENTS

## 列構成

| 列名 | 内容 |
|------|------|
| 分類 | 正常系/異常系/境界値など |
| 対象 | テスト対象の操作・入力 |
| 期待結果 | 期待される動作・結果 |

## 出力形式

### table（表形式）の場合

```markdown
| 分類 | 対象 | 期待結果 |
|------|------|----------|
| 正常系 | 有効なメールアドレスを入力 | 入力が受け付けられる |
| 異常系 | 空欄のまま送信 | エラーメッセージが表示される |
```

### list（箇条書き）の場合

```markdown
### 正常系
- [ ] 有効なメールアドレスを入力 → 入力が受け付けられる
- [ ] 有効なパスワードを入力 → 入力が受け付けられる

### 異常系
- [ ] 空欄のまま送信 → エラーメッセージが表示される
- [ ] 不正な形式のメールアドレス → バリデーションエラー
```

## テストケース作成のガイドライン

- **正常系**: 期待通りの入力・操作
- **異常系**: 不正な入力、エラーケース
- **境界値**: 最小値、最大値（該当する場合）

## 出力

1. マークダウン形式でテストケースを出力してください
2. コードブロックで囲まず、そのままコピペできる形式で出力してください
3. GitHub Issueにそのまま貼り付けられるようにしてください
