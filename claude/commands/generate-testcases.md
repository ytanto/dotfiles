# テストケースTSV生成

指定された機能「$ARGUMENTS」のテストケースを作成し、TSV形式でファイル出力してください。

## 出力形式

以下の列構成でTSV形式のファイルを生成してください：

```
ID	大分類	中分類	テスト観点	前提条件	操作手順	期待結果	確認結果
```

### 各列の説明

| 列名 | 内容 |
|------|------|
| ID | テストケースID（例: TC-001, TC-002） |
| 大分類 | 機能の大カテゴリ |
| 中分類 | 機能のサブカテゴリ |
| テスト観点 | このテストで何を検証するか |
| 前提条件 | テスト実行前に必要な状態・条件 |
| 操作手順 | 実行するステップ（番号付きリスト） |
| 期待結果 | 期待される動作・結果 |
| 確認結果 | 空欄（手動記入用） |

## TSV形式のルール（重要）

1. **列の区切り**: タブ文字（`\t`）で区切る
2. **行の区切り**: 改行で区切る
3. **セル内改行**: セル内に改行が必要な場合（操作手順など）は、セル全体をダブルクォート（`"`）で囲み、その中に実際の改行を入れる
   - 例: `"1. ユーザー名を入力する
2. パスワードを入力する
3. ログインボタンをクリックする"`
4. **ヘッダー行**: 必ず1行目にヘッダーを含める

## テストケース作成のガイドライン

### 網羅すべき観点
- **正常系**: 期待通りの入力・操作での動作
- **異常系**: 不正な入力、エラー時の動作
- **境界値**: 最小値、最大値、境界付近の値
- **権限**: 権限の有無による動作の違い（該当する場合）
- **状態遷移**: 状態の変化に関するテスト

### 操作手順の書き方
- 番号付きリストで記述（1. 2. 3. ...）
- 具体的なUI要素名を使用
- 入力値がある場合は明記

### 期待結果の書き方
- 具体的な画面状態・メッセージを記述
- 複数の確認ポイントがある場合は番号付きで列挙

## 出力

1. Writeツールを使って `testcases.tsv` というファイル名で現在のディレクトリに保存してください
2. 出力完了後、ファイルパスをユーザーに通知してください
3. テストケースの総数を報告してください

## 注意事項

- TSV形式は厳密に守ってください（スプレッドシートにそのままインポートできるように）
- 機能の理解が必要な場合は、関連するコードを読んでから作成してください
- テストケースは実用的で、実際にテスト実行可能なレベルの具体性を持たせてください
- IDは連番で良い
